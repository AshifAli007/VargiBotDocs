node_iot_ros_bridge_action_client module¶

This node is the Action client responsible for sending

goals/orders to the Action server. it also subscribes to MQTT topic /eyrc/vb/eTrzIron/orders to recieve orders online and than push this data in the form of goal to Action server.

class node_iot_ros_bridge_action_client.IotRosBridgeActionClient¶

It contains all the functions of an Action Client

responsible for sending goals to Action server.

on_transition(goal_handle)¶

Responsible for providing status of the goal/orders

Parameters: goal_handle – (ordersActionMsgGoal) details of goal
send_goal(priority, order)¶

as soon as an order is recieved online this function is

called which is responsible for sending this recieved goal to the server for further processing.

Parameters:
  • priority – (str)
  • order – (str) string of dict
Returns:

goal_handle
update_incoming_orders(priority, cost, order)¶

As soon as an order is recieved online this function is

called to update the incomingOrders sheet of IMS.

Parameters:
  • priority – (str)
  • cost – (int)
  • order – (dict) contain order details in dictionary
node_iot_ros_bridge_action_client.main()¶
node_iot_ros_bridge_action_client.on_connect(client, userdata, flags, rc)¶

As soon as this action client made connection with

mqtt this method is called which ensures that connection is established now we can recieve orders.

node_iot_ros_bridge_action_client.on_message(client, userdata, message)¶

As soon as an order is published on the topic

/eyrc/vb/eTrzIron/orders this function is called which than sends message as a goal/order to Action server for further processing. It also updates the incomingOrders sheet of IMS.
